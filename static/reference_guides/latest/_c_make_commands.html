CMake commands

Contains specific CMake commands to manage project, assembly informations and application properties.

These specific CMake commands are automatically added when you add find_package(xtd REQUIRED) to your CMakeLists.txt file.

Examples
cmake_minimum_required(VERSION 3.20)
project(my_project)
find_package(xtd REQUIRED)

xtd Specific CMake commands
  • Generic commands
    • choice_options provides a choice options for the user to select an option from options list.
  • Target properties commands
    • target_categories Specifies the application categories.
    • target_default_namespace specifies the the base namespace for files added to the project.
    • target_display specifies if application is displaying in GNOME or KDE menu.
    • target_icon specifies the icon file (.ico for Windows, icons for maOS and png or svg for linux) that you want to use as your program icon.
    • target_name specifies the name of the output file.
    • target_registered Specify the target is registered in the system.
    • target_startup specifies the entry point to be called when the application is load.
    • target_type specifies the type of target to build.
  • Assembly informations commands
    • assembly_company specifies a company name for the assembly manifest. Corresponds to AssemblyCompanyAttribute.
    • assembly_configuration specifies a configuration for the assembly manifest. Corresponds to AssemblyConfigurationAttribute.
    • assembly_copyright specifies a configuration for the assembly manifest. Corresponds to AssemblyConfigurationAttribute.
    • assembly_culture specifies the culture for the assembly manifest. Corresponds to AssemblyConfigurationAttribute.
    • assembly_description specifies an optional description for the assembly manifest. Corresponds to AssemblyDescriptionAttribute.
    • assembly_file_version specifies a version number that instructs the compiler to use a specific version for the Win32 file version resource. Corresponds to AssemblyFileVersionAttribute.
    • assembly_guid specifies an unique GUID that identifies the assembly. When you create a project, Visual Studio generates a GUID for the assembly. Corresponds to Guid.
    • assembly_product specifies a product name for the assembly manifest. Corresponds to AssemblyProductAttribute.
    • assembly_title specifies a title for the assembly manifest. Corresponds to AssemblyTitleAttribute.
    • assembly_trademark specifies a trademark for the assembly manifest. Corresponds to AssemblyTrademarkAttribute.
    • assembly_version specifies the version of the assembly. Corresponds to AssemblyVersionAttribute.
  • Build options commands
    • build_output_directory specifies the directory into which runtime target files should be built.
  • Install commands
    • add_install_include_directories Install include directories.
    • add_install_include_files installs include files.
    • install_component installs component.
    • install_package installs application MODULE_LIBRARY, STATIC_LIBRARY or SHARED_LIBRARY package.
    • install_name specifies the install name.
  • Project commands
    • add_include_directories Add include directory to current project.
    • add_packages adds, finds and loads settings from external required projects to current project.
    • add_projects adds child projects to parent project.
    • add_references adds references (libraries) to current project.
    • resource adds resources to current project.
    • resource_string adds resource string to current project.
    • setting adds a setting to current project.
    • add_sources adds source and header files to current project.
xtd Specific CMake variables
  • Target properties Variables
    • TARGET_CATEGORIES contains the application categories.
    • TARGET_DEFAULT_NAMESPACE contains default namespace use by application.
    • TARGET_ICON contains application icon file name without extension.
    • TARGET_NAME contains application name.
    • TARGET_REGISTERED contains boolean that specify if application is registered or not on the system.
    • TARGET_TYPE contains application type.
  • Assembly informations Variables
    • ASSEMBLY_COMPANY contains assembly company name.
    • ASSEMBLY_CONFIGURATION contains assembly configuration string.
    • ASSEMBLY_COPYRIGHT contains assembly copyright string.
    • ASSEMBLY_CULTURE contains assembly culture information.
    • ASSEMBLY_DESCRIPTION contains assembly description string.
    • ASSEMBLY_FILE_VERSION contains assembly file version number.
    • ASSEMBLY_GUID contains assembly guid string.
    • ASSEMBLY_PRODUCT contains assembly product string.
    • ASSEMBLY_TITLE contains assembly title string.
    • ASSEMBLY_TRADEMARK contains assembly trademark information.
    • ASSEMBLY_VERSION contains assembly version number.
  • Build options Variables
    • BUILD_OUTPUT_DIRECTORY contains build output directory.
    • BUILD_TYPE contains build type.
  • Install Variables
    • INSTALL_COMPONENTS contains install project components.
    • INSTALL_INCLUDE_DIRECTORIES contains install include directories.
    • INSTALL_INCLUDE_FILES contains install include files.
    • INSTALL_NAME contains install name.
  • Project Variables
    • PROJECT_APPLICATION_SETTINGS contains project application settings.
    • PROJECT_INCLUDE_DIRECTORIES contains include directories.
    • PROJECT_PACKAGES contains project packages.
    • PROJECT_PROJECTS contains solution or project projects.
    • PROJECT_REFERENCES contains project references.
    • PROJECT_RESOURCES contains project resources.
    • PROJECT_RESOURCE_STRINGS contains project resource strings.
    • ProjectSettingsSubSection contains project settings.
    • PROJECT_SOURCES contains project sources.
    • PROJECT_USER_SETTINGS contains project user settings.
  • System Variables
    • USER_DIRECTORY The path to user home.

Generic commands

choice_options

Provides a choice options for the user to select an option from options list.

Parameters
OPTIONthe option to set.
MESSAGEDescription message for the choice options.
DEFAULT_OPTIONThe default option.
...Other options.
Remarks
This method must be call before target_type().
This method is optional.
Examples
cmake_minimum_required(VERSION 3.20)
project(my_project)
find_package(xtd REQUIRED)
add_sources(my_project.cpp)
choice_options(TARGET_TYPE "Choose library dynamic, static, interface or module" DYNAMIC_LIBRARY STATIC_LIBRARY INTERFACE_LIBRARY MODULE_LIBRARY)
target_type(${TARGET_TYPE})

target_icon

Specifies the icon file (.ico for Windows, icons for maOS and png or svg for linux) that you want to use as your program icon.

Parameters
ARGV0the icon to set without extension. It can be empty.
Remarks
Do not specify the extension of file. It's operation system dependant.
If you write multiplatform application you must have the specified file in ico, icons and png or svg formats.
On Linux you can specify one icon format like icon.png or icon.svg (typically 512 x 512 pixels) or more icon format terminated by the number of pixels:
  • icon8.png (8 x 8 pixels)
  • icon16.png (16 x 16 pixels)
  • icon22.png (22 x 22 pixels)
  • icon24.png (24 x 24 pixels)
  • icon32.png (32 x 32 pixels)
  • icon36.png (36 x 36 pixels)
  • icon42.png (42 x 42 pixels)
  • icon48.png (48 x 48 pixels)
  • icon52.png (52 x 52 pixels)
  • icon64.png (64 x 64 pixels)
  • icon72.png (72 x 72 pixels)
  • icon96.png (96 x 96 pixels)
  • icon128.png (128 x 128 pixels)
  • icon192.png (192 x 192 pixels)
  • icon256.png (256 x 256 pixels)
  • icon512.png (512 x 512 pixels)
Call only once by project.
This method must be call before target_type().
This method is optional.
Examples
cmake_minimum_required(VERSION 3.20)
project(my_project)
find_package(xtd REQUIRED)
add_sources(my_project.cpp)
target_icon(my_icon)
target_type(CONSOLE_APPLICATION)

target_startup

Specifies the entry point to be called when the application is load.

Parameters
ARGV0the object that contains the entry point.
ARGV1the object definition file. (optional if you use TUnitMain_ or TUnitFrameworkMain_ entry point)
Remarks
Do not use this method of startup_(...) is present in your files or if you use standard c++ main() method.
Call only once by project.
This method must be call before target_type().
This method is optional.
Examples
cmake_minimum_required(VERSION 3.20)

target_type(GUI_APPLICATION)

target_type

Specifies the type of application to build.

Parameters
TARGET_TYPEType of application. Possible values :
  • CONSOLE_APPLICATION Console application
  • GUI_APPLICATION Gui application (Winforms)
  • TEST_APPLICATION Test application (TUnit)
  • MODULE_LIBRARY Module library
  • SHARED_LIBRARY Shared library (.dll or .so)
  • STATIC_LIBRARY Static library (.lib or .a)
  • INTERFACE_LIBRARY Interface library (header files only)
  • CUSTOM_TARGET Target with no output.
Remarks
This method is required except container project like Solution.
This method must be the last method.
Call only once by project.
Examples
cmake_minimum_required(VERSION 3.20)
project(my_project)
find_package(xtd REQUIRED)
add_sources(my_project.cpp)
add_references(xtd.drawing xtd.drawing.native)
target_type(CONSOLE_APPLICATION)
